# Rule: Innovation & Modernity 🚀

## Goal

Encourage the use of cutting-edge technologies and patterns while maintaining project stability.

## Guidelines

- **Experimental Features**: If a new React 19 feature or a modern API (like Web Capabilities) can simplify code or improve UX, feel free to propose its implementation.
- **Continuous Improvement**: Don't stick to a pattern just because it's already there. If you find a more efficient or elegant way to solve a problem, suggest it in the `implementation_plan.md`.
- **Performance First**: Innovation should ideally lead to better performance. Use modern tools (like Bun features or Vite optimizations) to keep the app fast.
- **Open Standards**: Prioritize standard web APIs over heavy libraries when possible to ensure long-term sustainability.
